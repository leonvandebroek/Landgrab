## Summary

Fixed three interconnected bugs preventing beacon cone tiles from rendering visible state and heads-up responsiveness:

1. **Bug A:** Debug heading (Q/E) not forwarded to overlay layer
2. **Bug B:** `BeaconHeading` unconditionally nulled on every heartbeat
3. **Bug C:** Beacon cone tiles still rendered as Hidden (no troop count reveal)

**Build result:** ✅ 293 modules, clean lint, zero errors

---

## Technical Resolution

### Bug A — Heading field mismatch (cone never reactive to Q/E)

**Root cause:** `GameMap.tsx` passed raw compass sensor heading to overlay. Debug heading key press updates (Q/E) only existed in local `debugCompassHeading` state, never reaching `AbilityOverlayLayer` which still read stale `myPlayer.beaconHeading` from server.

**Fix:**
- `GameMap.tsx` line 896: Forward effective heading as `compassHeading={debugCompassHeading ?? compassHeading}`
- `AbilityOverlayLayer.tsx`: Use effective heading in `beaconState` and `beaconScanHexes` computation via `compassHeading ?? myPlayer.beaconHeading`
- Added `compassHeading` to dependencies

### Bug B — BeaconHeading wiped on every location heartbeat

**Root cause:** `GameplayService.UpdatePlayerLocation` contained logic that **always** nulled `BeaconHeading` when no compass sensor was available:
```csharp
player.BeaconHeading = player.CurrentHeading.HasValue
    ? HexService.NormalizeHeading(player.CurrentHeading.Value)
    : null;   // ← Wipes heading on every heartbeat without compass
```

Any heartbeat without heading (device missing compass, before Q/E pressed) destroyed the heading set by `ActivateBeacon`.

**Fix:** Only update `BeaconHeading` when heading is present:
```csharp
if (player.CurrentHeading.HasValue)
{
    player.BeaconHeading = HexService.NormalizeHeading(player.CurrentHeading.Value);
}
// Preserve existing BeaconHeading when no heading provided
```

### Bug C — Beacon cone tiles show Hidden (no troop count reveal)

**Root cause:** `tricorderTileState.deriveTileState` unconditionally returned hidden state for `visibilityTier === 'Hidden'` tiles before checking if they were beacon-scanned. Server sends full tile data for beacon cone hexes but frontend discarded it.

**Fix:**
- Added `beaconConeHexKeys: ReadonlySet<string>` to `gameplayStore`
- `AbilityOverlayLayer` syncs cone hexes into store via `setBeaconConeHexKeys`
- `tricorderTileState`: Override `isHidden`/`isRemembered` to false when hex is in beacon cone, allowing full Visible rendering
- `HexTile.tsx` and `TileInfoCard.tsx` pass `beaconConeHexKeys` to `deriveTileState`
