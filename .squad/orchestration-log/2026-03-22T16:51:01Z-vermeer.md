## Key Design Decisions

### Staleness Thresholds
- **Threshold:** 120s hardcoded as `STALENESS_FADING_THRESHOLD_MS = 120_000` in tricorderTileState.ts
- **Rationale:** Matches game dynamics spec. If future reconfiguration needed, can wire through DeriveTileStateParams.

### Own/Ally Tile Gating
- **Behavior:** Amber treatment only applies to remembered enemy tiles (visibilityTier === 'Remembered')
- **Rationale:** Own/ally tiles are always Visible from server perspective, so stalenessTier never returns non-'live' values for them.

### Missing Timestamp Fallback
- **Behavior:** When `lastSeenAt` absent, `computeStalenessTier` returns 'stale' (no fading tier)
- **Rationale:** Matches spec instruction: "if no timestamp available, treat as stale"
