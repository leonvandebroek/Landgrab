### P0: Defender combat feedback
- **Issue:** Attacker receives full `CombatResult` dialog; defender gets nothing when attack is repelled.

### P0: Alliance visibility in players panel
- **Issue:** Alliance disappeared from player list when no players matched (data race / transient state mismatch).
- **Fix implemented:**
  - Removed `if (alliancePlayers.length === 0) return null;` guard in `PlayingHud.tsx`.
  - All alliances from `state.alliances` now always render with name, color, territory count.
  - Added fallback for orphan `allianceId` references (player's alliance ID doesn't exist in state).
- **Verification:** `npm run build` clean; no TypeScript errors.

### P1: Troop pickup carrying total
- **Issue:** Pickup success message showed count added but not final carrying total; feedback incomplete.
- **Fix implemented:**
  - `useGameActionsGameplay.ts` `handleConfirmPickup` now reads `previousCarried` via `useGameStore.getState()` inside `.then()` callback (avoids stale closure).
  - Computes optimistic `newCarried = previousCarried + count`.
  - Updated i18n keys: `game.mapFeedback.pickedUp_one/other` → `'+{{count}} troop(s) picked up · Carrying: {{carrying}}'` (EN + NL).
- **Pattern:** For stale-closure-safe reads in async callbacks, use `useGameStore.getState()` rather than closure-captured props.
- **Verification:** `npm run build`, `npm run lint` pass.
