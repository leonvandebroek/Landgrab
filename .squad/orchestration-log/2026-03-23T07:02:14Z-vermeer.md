### Design Rationale

- Scout cone always-on removes toggle confusion; passive role contribution is clear
- Share Intel CTA makes team coordination explicit
- Cooldown timer shows Scout when next share is available
- Server-side cone computation prevents client drift (backend De Ruyter computes authoritative geometry)

### Backward Compatibility

- Non-role mode games retain full beacon toggle + activate/deactivate card unchanged
- `BeaconCard.tsx` preserved for non-Scout beacon players
- All role-specific logic gated on `playerRolesEnabled && player.role === 'Scout'`

### Pattern Established

Optimistic UI for ability actions: call state mutation first, await server, revert on explicit `false`. Consumer memos must check both server flags and local `abilityUi` state to avoid broadcast-wait windows.
