*Date: 2026-03-25*

## Summary
Landgrab squad delivered critical security hardening (Phase 1) and structural refactoring (Phase 2). Phase 1 fixed 9 vulnerability/resilience issues: rate limiter, database retry, null guards, CSP headers, CORS isolation, FK shadow column elimination, and documentation corrections. ## Architectural Decisions (Phase 2 Design)
7 binding decisions merged into `.squad/decisions.md` (#28–34):
1. **Abstract base class** — `RoleAbilityServiceBase` carries shared helpers
2. **Explicit delegation** — No runtime role→service dictionary
3. **Four services** — Commander, Scout, Engineer, Shared
4. **RoleProgressService** — Extracted from `GameplayService` for fort/sabotage/demolish tracking
5. **Registry metadata-only** — Frontend card components statically imported
6. **AbilityCardProps contract** — All cards accept `{ myUserId, invoke }`
7. **makeHandler factory** — Replaces 19 duplicated try/catch blocks
