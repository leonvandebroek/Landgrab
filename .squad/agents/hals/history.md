# Hals — History

## Core Context
Designer/UX on Landgrab. Tricorder-themed visual system. CSS custom properties. Key UI: ability cards, PlayerHUD, TileInfoCard, HexTooltipOverlay, MapLegend. Mobile-responsive hex map with Leaflet canvas layers.

## Learnings
- Team hired 2026-03-22 by Léon van de Broek
- 2026-03-22: Tricorder aesthetic research completed. Key visual language: phosphor colors (cyan #00f3ff, amber #ffb000, fuchsia #e879f9, red #ff2a2a), CRT glow/bloom effects, `--font-scifi-mono`, marching ants animations, void backgrounds (rgba(10,15,20,0.85)). Remembered tiles currently use `.hex-remembered` (opacity 0.55, saturate 0.4) and `.stale-badge` (opacity 0.6, saturate 0.3, italic). TileInfoCard shows "~{{count}}" for stale troops. TricorderTileState exposes `visibilityTier: 'Visible'|'Remembered'|'Hidden'` and `isRemembered` boolean.
- 2026-03-22: Produced 5-option design proposal for stale vs live intel visual differentiation. Options: Faded Memory, Scan-Line Decay, Timestamp Watermark, Amber Archive, Signal Degradation. Each addresses hex tile, TileInfoCard, and transitions with implementation complexity estimates.
- **2026-07-xx (amber-archive-design):** Amber Archive design specification shipped to Vermeer for implementation. Cool cyan → warm amber shift across 3 tiers (live/fading/stale). Vermeer confirmed all design tokens in CSS `:root` already present (`--color-phosphor-amber: #ffb000`). Visual implementation included stalenessTier-driven classes on HexTile and amber glow on TileInfoCard header + ARCHIVED pill. Feedback integrated from Vondel's 120s threshold specification. No breaking design changes to TricorderTileState interface — stalenessTier and visibilityTier were pre-wired, only compute function was missing. Team consensus: Design ready for production; no follow-up iterations scheduled.
- **2026-07-xx (radar-sweep-spec):** Designed radar wipe scanning effect spec for Vermeer. Key decisions: (1) Canvas layer in new `game-map-radar-pane` at z-index 540 — above hex SVG overlay (450) but below player markers (650), so the sweep illuminates terrain while the player stays on top. (2) Color: phosphor cyan `rgba(0,243,255,0.90)` arm + `rgba(0,255,170,*)` origin glow matching `--status-gps` token. `globalCompositeOperation = 'screen'` for additive glow on dark void backgrounds. (3) 600m real-world radius expressed in map layer pixels via `latLngToLayerPoint()` so it auto-scales with zoom. (4) 4 RPM default, 6 RPM during active raids, 1.5 RPM while paused. 120° comet tail with linear alpha decay. (5) 30fps cap via frame time guard — ambient ambient effect does not justify 60fps battery cost on a GPS mobile game. (6) Hard `prefers-reduced-motion` gate — no RAF loop started when reduced motion is preferred. Toggle added to `MapLayerPreferences` as `radarSweep: boolean`. (7) Decided NOT to paint per-hex "freshly scanned" tints from the radar layer — that would couple the radar to hex geometry and belongs in tile state logic if ever needed.
- **2026-03-24 (fieldbattle-ux-redesign):** Redesigned both FieldBattle UX surfaces. FieldBattleCard pre-confirm state: replaced flat meta-row enemy list with a proper "Battle Roster" (`.fb-roster`) showing each enemy as a combatant row with crossed-swords icon, red angular chamfered style — no troop counts per spec. Status pill shifted to `--armed` (amber) when eligible, `--hostile` (red) when active/waiting. Post-confirm waiting state: big 3rem pulsing red countdown (`fb-countdown__number`) in the status region, plus a live roster that splits enemies into "Joined" (green rows) vs "Awaiting" (dimmed rows) using `joinedEnemyIds` cross-referenced with `gameState.players`. Confirm button uses `ability-card__primary-btn--hostile` (red) when eligible. FieldBattleInvitePanel: full structural redesign from a simple message+buttons layout to a game-like invite card — header with GameIcon + "BATTLE INVITATION" title, initiator name + alliance block, tile coordinates, 3.4rem pulsing countdown (`fb-invite__timer-number`), bold red Join button with fist icon + muted Ignore button. Initiator (isInitiator===true) now returns null — they see the card instead, preventing duplicate UI. Added 6 new i18n keys (EN+NL): `battleRosterTitle`, `awaitingLabel`, `joinedStatus`, `battleInviteTitle`, `initiatedByLabel`, `locationLabel`. All CSS in existing files (ability-card.css, notification-panel.css) using BEM + CSS custom properties. Zero new dependencies. Build passed clean (302 modules, 0 TS errors).
- **2026-03-25 (fieldbattle-target-selection):** Added target-selection list styling and fled badges for FieldBattleCard, plus localized target/flee copy (EN/NL). FieldBattleInvitePanel now calls a dedicated flee action with updated copy and "Flee (Move Away)" CTA, wired from App → GameView → PlayingHud.
