6. THREAD SAFETY ARCHITECTURE
═══════════════════════════════════════════════════════════════════════════════════

┌─ SYNCHRONIZATION MODEL ────────────────────────────────────────────────────┐
│                                                                              │
│  GameRoom (holds mutable state)                                             │
│  ├─ public object SyncRoot { get; } = new object()                         │
│  ├─ public GameState State { get; set; }                                   │
│  ├─ public ConcurrentDictionary<string, string> ConnectionMap              │
│  └─ public object ConnectionUpdateLock                                     │
│                                                                              │
│  All GameService methods follow:                                            │
│  ┌──────────────────────────────────────────────────────────────────────┐ │
│  │ public (GameState? state, string? error) MethodName(...)             │ │
│  │ {                                                                    │ │
│  │   var room = GetRoom(roomCode);  // No lock needed (lookup)          │ │
│  │   if (room == null) return (null, "error");                          │ │
│  │                                                                      │ │
│  │   lock (room.SyncRoot)  // ◄── CRITICAL SYNCHRONIZATION POINT       │ │
│  │   {                                                                  │ │
│  │     // All mutations here are atomic                                │ │
│  │     room.State.Players.Add(...);                                    │ │
│  │     room.State.Grid[hex].Troops += 5;                               │ │
│  │     QueuePersistence(room, snapshot);                               │ │
│  │     return (SnapshotState(room.State), null);                       │ │
│  │   }                                                                  │ │
│  │ }                                                                    │ │
│  └──────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│  Background Services (TroopRegen, RandomEvent, Mission):                    │
│  ┌──────────────────────────────────────────────────────────────────────┐ │
│  │ protected override async Task ExecuteAsync(CancellationToken ct)     │ │
│  │ {                                                                    │ │
│  │   using var timer = new PeriodicTimer(TimeSpan.FromSeconds(30));   │ │
│  │   while (!ct.IsCancellationRequested &&                             │ │
│  │          await timer.WaitForNextTickAsync(ct))                      │ │
│  │   {                                                                 │ │
│  │     try {                                                           │ │
│  │       using var scope = scopeFactory.CreateScope();               │ │
│  │       var gameService = scope.GetRequiredService<GameService>();  │ │
│  │                                                                     │ │
│  │       foreach (var roomCode in gameService.GetPlayingRoomCodes()) │ │
│  │       {                                                             │ │
│  │         var room = gameService.GetRoom(roomCode);                 │ │
│  │         if (room == null) continue;                               │ │
│  │                                                                    │ │
│  │         lock (room.SyncRoot)  // ◄── DIRECT MUTATION WITH LOCK     │ │
│  │         {                                                          │ │
│  │           room.State.Grid.Values.ForEach(hex =>                   │ │
│  │             hex.Troops += 1);  // Direct mutation here             │ │
│  │           gameService.ProcessDuelExpiry(room);                    │ │
│  │         }                                                          │ │
│  │       }                                                             │ │
│  │     }                                                               │ │
│  │     catch(Exception ex) { logger.LogError(...); }                  │ │
│  │   }                                                                 │ │
│  │ }                                                                    │ │
│  └──────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│  ✓ CONSTRAINT: Refactored services MUST maintain this lock discipline      │
│    - No new locks for individual services                                   │
│    - Use room.SyncRoot consistently                                         │
│    - Background services continue direct mutation (performance critical)    │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

7. IMPLEMENTATION PHASES (Dependency Order)
═══════════════════════════════════════════════════════════════════════════════════

  PHASE 1: Foundation
  ─────────────────────────────────────────────────────────────────────────
    Extract: RoomConnectionService
    Tests: Room creation, joining, disconnection
    Risk: LOW - Lowest level, no dependencies on other new services
    
    └─> BLOCKS: Phase 2,3,4,5,6,7,8 (all depend on this)

  PHASE 2: Utilities  
  ─────────────────────────────────────────────────────────────────────────
    Extract: GameStateManagementService
    Tests: Snapshots, visibility, fog of war
    Risk: LOW - Utility only, no state mutation
    Dependencies: NONE (isolated)
    
    └─> CRITICAL for background services (TroopRegen, etc.)

  PHASE 3: Duel System
  ─────────────────────────────────────────────────────────────────────────
    Extract: DuelManagementService
    Tests: Duel creation, resolution, expiry
    Risk: MEDIUM - Complex duel logic
    Dependencies: RoomConnectionService
    
    └─> Required by TroopRegenerationService

  PHASE 4: Lobby Configuration
  ─────────────────────────────────────────────────────────────────────────
    Extract: GameLobbyService
    Tests: Map config, rules, alliances
    Risk: MEDIUM - 20 validation-heavy methods
    Dependencies: RoomConnectionService, GlobalMapService, TerrainFetchService
    
    └─> Updates GameHub (~20 method calls)

  PHASE 5: Game Startup
  ─────────────────────────────────────────────────────────────────────────
    Extract: GameStartService
    Tests: Game initialization, starting conditions
    Risk: MEDIUM - Complex grid setup
    Dependencies: RoomConnectionService, GameLobbyService, HexService
    
    └─> Updates GameHub (~5 method calls)

  PHASE 6: Gameplay (CRITICAL PATH)
  ─────────────────────────────────────────────────────────────────────────
    Extract: GameplayService (★ LARGEST - 1,200 LOC)
    Tests: Movement, combat, troop placement, reinforcements
    Risk: HIGH - Complex combat logic, 400 LOC UpdatePlayerLocation
    Dependencies: RoomConnectionService, HexService, DuelManagementService
    
    └─> Updates GameHub (~12 method calls)
    └─> Updates TroopRegenerationService (AddReinforcementsToAllHexes)

  PHASE 7: Remaining Abilities & Admin
  ─────────────────────────────────────────────────────────────────────────
    Extract: 
      • CopresenceAbilityService (4 methods, 300 LOC)
      • HostageManagementService (2 methods, 80 LOC)
      • EventManagementService (1 method, 100 LOC)
      • AdminHostService (5 methods, 200 LOC)
    
    Dependencies: Above services
    Tests: Abilities, hostage capture, event logging
    Risk: MEDIUM - Less critical paths
    
    └─> Updates GameHub (~15 method calls total)

  PHASE 8: Integration & Validation
  ─────────────────────────────────────────────────────────────────────────
    Verify:
      • GameHub functional (all 104 calls routed)
      • TroopRegenerationService (background tick)
      • RandomEventService (event generation)
      • MissionService (mission tracking)
      • Thread safety (no race conditions)
      • Performance (no regressions)
    
    Cleanup:
      • Remove old GameService if not used as facade
      • Update documentation
      • Update DI configuration
